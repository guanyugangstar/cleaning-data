# EDA 分析提示词模板

  

> 出处：[SKILL.md §8.2](../SKILL.md#82-eda-分析提示词)

  

```

你是一个数据质量专家。请分析以下 EDA 报告，检查脚本可能遗漏的异常。

  

=== EDA 报告 ===

{EDA报告内容}

  

请重点检查以下异常信号：

  

1. **数据完整性问题**

   - 数据列数是否过少（如只有1列）？

   - 数据行数是否合理？

  

2. **统计异常**

   - std 为 NaN 或 0（可能数据类型错误或数据全相同）

   - mean 为 NaN

   - 偏度/峰度为 NaN

  

3. **业务逻辑异常**

   - 分类列的唯一值数量是否合理（太多或太少）

   - 重复行比例是否异常高

   - 缺失率是否突然变化

  

4. **分类编码异常**

   - 分类编码是否混乱（如同一类别有多种表示方式："男"/"M"/"male"）

   - 编码值是否出现乱码或异常字符

   - 编码前后不一致（如"初中"和"初"混用）

5. **分类列输入变体**（重要！）

   - 检查 `categorical_distributions` 中是否有**低频值**（如出现1次的值）

   - 低频值可能是输入错误（如"住建局限"应为"住建局"、"税物局"应为"税务局"）

   - 如发现输入变体，需在 `format_standardization.value_correction` 中配置纠正规则

  

6. **类型判断错误**

   - 数值列的 std 为 0（可能是分类数据误判为数值）

   - 日期列检测是否遗漏

  

请输出：

```
